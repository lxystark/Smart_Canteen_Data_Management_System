## 三、难度分析维度

> 指导问题：请逐项评估项目的技术难度、工程难度和资源难度

### 3.1 技术难度 ⭐⭐⭐

| 难度点 | 具体内容 | 评估 |
|-------|---------|------|
| **时序预测算法** | LSTM/Transformer 实现复杂度 | 中高 |
| **特征工程** | 外部数据（天气、课程表）融合 | 中 |
| **模型优化** | 超参数调优、防止过拟合 | 中 |
| **数据预处理** | 脏数据清洗、缺失值处理 | 低中 |
| **实时预测** | 模型部署、在线推理 | 高 |

**技术栈要求：**
```
Python 3.x
├── pandas / numpy      (数据处理)
├── scikit-learn        (机器学习)
├── TensorFlow/PyTorch  (深度学习)
├── statsmodels         (ARIMA)
├── matplotlib/plotly   (可视化)
└── Flask/FastAPI       (可选：Web服务)
```

### 3.4 风险评估 ⚠️

| 风险类型 | 风险描述 | 应对策略 |
|---------|---------|---------|
| **数据不足** | 无法获取足够的真实数据 | 使用模拟数据 + 公开数据集 |
| **模型失效** | 预测准确率不达标 | 降低预期，多尝试几种模型 |
| **范围蔓延** | 功能做太多做不完 | MVP优先，先做核心功能 |
| **技术选型** | 选了太难的技术栈 | 从简单算法开始，逐步升级 |
